sleep 与wait的区别
 两者都是让线程暂停一段时间 但本质区别是一个是线程的运行状态控制,一个是线程之间的通讯问题

1.sleep在java.lang.Thread类中 
wait 在java.lang.Object类中

2.sleep是使一个线程睡眠,参数可以指定一个睡眠时间,在睡眠期间
线程不在执行,但线程也不会释放所占用的系统资源,而wait是进程的运行机制的
一种状态,当执行wait时,该线程会释放所占有的系统资源,进入就绪状态等待下一次处理机分配运行.在这点上sleep虽然和wait在效果上相似当时sleep后线程仍然占用系统资源,wait不在占用系统资源

3.sleep没有释放锁,其他线程无法使用临界资源而wait方法释放了锁,在等待期间让给其他线程使用
sleep必须捕获异常,wait不用捕获异常
4.wait notify notifyall 只能在同步控制方法或同步控制块中使用,而sleep可以在任何地方使用synchronizd(x)

java中实现线程的方式,都是什么关键字修饰同步方法 stop和suspend为何不推荐使用

有Thread类和Runnable接口

用synchronized关键词修饰同步方法  synchronized
反对使用stop是因为他不安全,会释放由线程获取的所有锁定,如果对象处于一种不连贯状态,那么其他线程不能在那种状态下检查和修改他们,结果很难查出问题的所在.
suspend容易产生死锁,使用挂起操作,目标线程会停下来,但仍然持有之前获得的锁定.此时其他任何线程都不能访问锁定的资源,除非被挂起的线程恢复运行使用


==========================同步和异步的异同
线程的执行时run()还是start()
如果数据将在线程中共享,正在写入的数据以后会被其他的线程用到,就应该使用同步

当程序在某个对象上调用了一个需要花费较长时间的方法,而又不希望程序一直等待返回,继续执行提高效率的时候,就使用异步


启动线程使用start()
run()是线程内部有的方法,在线程执行时,线程会自动调用run()方法
